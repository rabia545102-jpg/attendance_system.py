Q1 Student Result Analyzer Easy

variablesint / float / str if-elif-elsefor loop listbasic function

Problem Statement

You are given the marks of a student in 5 subjects. Write a Python program that:

  1. Stores the student's name (string) and roll number (integer) in variables.
  2. Takes 5 subject marks as a list of floats.
  3. Calculates the total and average marks.
  4. Assigns a grade using the following rules:
    • average >= 90 → Grade A
    • average >= 75 → Grade B
    • average >= 60 → Grade C
    • average >= 40 → Grade D
    • else → Fail
  5. Also prints which subjects the student scored below 40 in (use a loop).
  6. Wrap all logic inside a function analyze_result(name, roll, marks).
Sample Input
name = "Aarav"
roll = 101
marks = [88.5, 35.0, 76.0, 92.5, 48.0]
Expected Output
Student: Aarav (Roll: 101)
Total: 340.0, Average: 68.0
Grade: C
Subjects below 40: Subject 2

Q2 Library Book Management System Medium

dictionarylist settuple while loopnested conditions functions

Problem Statement

You are building a small Library System. Implement the following using the right data structure for each requirement:

  1. Store the library catalog as a dictionary where key = book_id (int) and value = tuple (title, author, year). (Why tuple? Because book details should be immutable.)
  2. Maintain a list called borrowed_books to track currently issued book IDs (insertion order matters).
  3. Maintain a set called members to store unique member IDs (no duplicates allowed).
  4. Write the following functions:
    • add_book(catalog, book_id, title, author, year)
    • borrow_book(catalog, borrowed_books, book_id) — only if book exists & not already borrowed.
    • return_book(borrowed_books, book_id)
    • register_member(members, member_id) — should silently ignore duplicates.
    • show_available(catalog, borrowed_books) — print all books that are NOT in borrowed_books.
  5. In main(), demonstrate adding 4 books, registering 3 members (try adding the same member twice), borrowing 2 books, returning 1, and finally displaying available books.
Why each structure?
  • Dict → fast lookup of books by ID.
  • Tuple → book details shouldn't change after creation.
  • List → borrowed history with order.
  • Set → automatically prevents duplicate members.

Save your code as: q2_library_system.py

Q3 Shopping Cart with Default & Mutable Pitfall Hard / Concept-heavy

default parameters mutable vs immutable function arguments list / dict loops

Problem Statement

This question tests the trickiest topic we covered — mutable default arguments & pass-by-reference behavior.

Part A — Spot the Bug

The following function has a hidden bug. Predict the output and explain why:

def add_item(item, cart=[]):
    cart.append(item)
    return cart

print(add_item("apple"))
print(add_item("banana"))
print(add_item("milk", cart=["bread"]))
print(add_item("eggs"))

Part B — Fix It

Rewrite add_item the correct way so that calling it without a cart starts with a fresh empty list every time.

Part C — Build the Cart

Now build a complete shopping cart program:

  1. Function create_cart(owner, discount=0) → returns a dict like {"owner": owner, "items": [], "discount": discount}. Here discount is an immutable default — safe.
  2. Function add_to_cart(cart, name, price, qty=1) → appends {"name": name, "price": price, "qty": qty} to cart["items"].
  3. Function update_price(price_tuple, new_price) → attempt to modify a tuple element. What happens? Explain in a comment why tuples are immutable.
  4. Function calculate_total(cart) → loops over items, sums price * qty, applies discount %, returns final total.
  5. Demonstrate calling these functions for two different customers and show their independent carts (proves the Part B fix works).
Expected behavior:
• Two carts created separately must NOT share items.
• Modifying one list inside one cart must not affect the other.
• Trying to change a tuple should raise TypeError.

Save your code as: q3_shopping_cart.py

Discussion Points (write your answer as comments at the bottom of the file)

  • Why is discount=0 safe but cart=[] dangerous?
  • What is the difference between rebinding and mutating?
  • Which of these are mutable? — list, tuple, dict, set, str, int
  • When you pass a list into a function and modify it, do changes reflect outside? Why?
